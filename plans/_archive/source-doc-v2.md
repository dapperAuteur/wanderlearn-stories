# **Performance Budgeting for WebXR 360 Tours**

This table defines the "Hard Limits" for A-Frame/Three.js scenes. If a design exceeds these, the WebXR Architect must flag it for immediate optimization to prevent mobile browsers from crashing.

| Metric | Target (Mobile WebXR/A-Frame) | Target (Desktop WebXR) | Priority |
| :---- | :---- | :---- | :---- |
| **Frame Rate** | 60 FPS (Stable) | 60 FPS (Stable) | Critical |
| **Draw Calls** | \< 100 per frame | \< 250 per frame | High |
| **Triangles** | \< 500,000 (Visible) | \< 1,500,000 (Visible) | Medium |
| **Texture Size** | 1024px Max, 2048px (Hero only) | 2048px Max, 4096px (Hero only) | High |
| **Compression** | .glb with Draco/KTX2 compression | .glb or .gltf | High |
| **Memory (RAM)** | \< 1GB (iOS Safari Crash Limit) | \< 2GB | Critical |
| **Audio** | Highly compressed .mp3 or .ogg | High-quality .mp3 | Low |

### **Architect Enforcement Logic:**

1. **The 3D Asset Mandate:** All 3D models must be exported as .glb (GL Transmission Format). This is the "JPEG of 3D" and is native to the web.  
2. **DOM Efficiency:** Avoid creating and destroying HTML \<a-entity\> tags constantly. Instead, use "Object Pooling" (hiding and showing existing entities) to prevent garbage collection stuttering.  
3. **The "Baked" Mandate:** Real-time lighting calculates on the CPU/GPU and destroys web performance. All shadows and lighting must be "baked" into the textures in a program like Blender before exporting to the web.

# **Story Engine Architecture: Making Books Modular**

**Objective:** To allow the Project Owner to add new public-domain books to the app by swapping JSON files and 3D models, without rewriting the core JavaScript logic.

## **1\. The "Theater" Concept**

The HTML file (index.html) acts as the Stage. The 3D environments (The Woods, The Hall) are generic .glb files loaded dynamically. The story itself is injected via a JSON configuration.

## **2\. Story Configuration Schema (JSON)**

The dev team must build a curriculum-loader.js component to parse a story\_config.json file.

{  
  "book\_id": "alice\_wonderland",  
  "title": "Alice's Adventures in Wonderland",  
  "hubs": \[  
    {  
      "hub\_id": 1,  
      "environment\_model": "assets/models/vertical\_tunnel.glb",  
      "story\_audio\_clip": "assets/audio/alice\_ch1.mp3",  
      "text\_excerpt": "Down, down, down. Would the fall never come to an end?",  
      "available\_easter\_egg\_slots": 3  
    }  
  \]  
}

## **3\. Developer Deliverables**

To make this work in a WebXR environment, the development team must provide:

1. **Dynamic Asset Loading:** A script that reads the environment\_model path from the JSON and uses JavaScript to update the gltf-model attribute of the \<a-scene\> background entity.  
2. **Standardized Content Management:** Instead of a complex Unity Editor tool, the Project Owner will simply manage standard JSON files. The Dev Team must ensure the JS gracefully handles missing properties or typos in the JSON.

# **Learning Analytics & Spaced Repetition System**

**Objective:** To transform the app from a passive tour into an adaptive learning engine that recognizes user weaknesses and applies spaced recall to help them improve.

## **1\. The Analytics Loop**

Every interaction with an Easter Egg sends a telemetry event to Firebase Firestore.

* **Data Logged:** UserID, Subject (e.g., Math\_Addition), Difficulty\_Level, Success (Boolean), Time\_Taken.

## **2\. The Spaced Recall Algorithm (The "Flashcard" Engine)**

The CurriculumManager.cs does not randomly select Easter Eggs. It uses a weighted algorithm based on the user's history.

* **The Weighting System:**  
  * If the user gets a "Sight Word" correct, the weight of Literacy drops.  
  * If the user gets an "Addition" question wrong, the weight of Math increases by 50%.  
* **Dynamic Spawning:** When the user enters Hub 2, the engine checks the weights. If Math is high, it pulls 3 Math facts from the JSON and 0 Literacy facts, forcing the user to practice what they struggle with to progress.

## **3\. Gamification of Weaknesses**

To prevent frustration when practicing difficult subjects, the system offers higher rewards:

* "You found a Rare Math Gem\! These only appear for the bravest scholars\!"  
* Earning stickers specifically tied to improving a previously failed subject.
