# Wanderlearn Stories — public docs

Public-facing documentation for admins, teachers, illustrators,
narrators, and future contributors. These render plain Markdown; a
future iteration may serve them via a `/docs` route on the site.

| File | Audience |
|------|----------|
| [easter-eggs.md](easter-eggs.md) | Anyone authoring or reviewing an easter egg — the smallest unit of learning content in the product |

## What's not in here

- **Internal planning** lives in `plans/` (not in the repo; ask BAM
  for access). That's where roadmaps, tier scoping, performance
  budgets, COPPA design, and other in-flight decisions sit.
- **Code documentation** for engineers lives next to the code itself
  (TSDoc / inline comments). If you're touching the codebase, start
  with [`README.md`](../README.md) and [`AGENTS.md`](../AGENTS.md) at
  the repo root.

If you're a teacher reviewing standards alignment or a parent
wondering what data we collect, this directory is the right entry
point. If you're trying to author a whole new book, the easter-eggs
guide covers the smallest unit; the broader story-authoring guide
will land alongside as the schema stabilizes.
